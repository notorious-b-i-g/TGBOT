# TGBOT

Telegram-бот для управления заказами и задачами с ролями клиент / исполнитель / администратор. Данные хранятся в удалённой базе MySQL, доступ к которой осуществляется через SSH-туннель, а отчётность выгружается в Google Sheets.

## О проекте

Бот автоматизирует приём и сопровождение заказов: клиент оформляет заявку, исполнитель берёт её в работу и завершает, а администратор контролирует процесс и формирует отчётность. Логика диалогов построена на конечных автоматах (FSM) aiogram, благодаря чему многошаговые сценарии (оформление заказа, редактирование, завершение) остаются предсказуемыми и устойчивыми.

## Стек

- **Python** — основной язык
- **aiogram 2.13** (aiogram 2.x) — Telegram Bot API, FSM, middleware
- **aiomysql** — асинхронный доступ к MySQL (пул соединений)
- **sshtunnel + paramiko** — подключение к удалённой БД через SSH-туннель
- **gspread + google-auth** — интеграция с Google Sheets через сервисный аккаунт
- **python-dotenv** — конфигурация через `.env`

## Возможности

- **Роли пользователей:**
  - *Клиент* — оформление и отслеживание заказов;
  - *Исполнитель* — взятие заказов в работу, отчёт о выполнении, завершение;
  - *Администратор* — управление заказами и пользователями, контроль процесса.
- **Управление заказами** — создание, редактирование, проверка и завершение задач через пошаговые FSM-сценарии.
- **Отчётность в Google Sheets** — экспорт и формирование отчётов по заказам в таблицы Google.
- **MySQL через SSH-туннель** — безопасное подключение к удалённой базе данных без прямого внешнего доступа к порту СУБД.
- **Фоновые задачи** — автоматическая очистка устаревших заказов.

## Запуск

```bash
# 1. Клонировать репозиторий
git clone https://github.com/notorious-b-i-g/TGBOT.git
cd TGBOT

# 2. Создать и активировать виртуальное окружение
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Подготовить конфигурацию
cp .env.example .env            # Windows: copy .env.example .env
# затем заполнить значения в .env

# 5. Запустить бота
python bot.py
```

Для интеграции с Google Sheets потребуется файл сервисного аккаунта Google и доступ к целевой таблице.

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните реальными значениями. Файл `.env` в репозиторий не попадает (см. `.gitignore`).

| Переменная | Описание |
|------------|----------|
| `API_TOKEN` | Токен Telegram-бота (получается у [@BotFather](https://t.me/BotFather)) |
| `DB_HOST` | Хост MySQL (локальный, так как используется туннель; по умолчанию `127.0.0.1`) |
| `DB_PORT` | Порт MySQL (по умолчанию `3306`) |
| `DB_USER` | Пользователь базы данных |
| `DB_PASSWORD` | Пароль пользователя базы данных |
| `DB_NAME` | Имя базы данных |
| `SSH_HOST` | Хост SSH-сервера для туннеля |
| `SSH_PORT` | Порт SSH (по умолчанию `22`) |
| `SSH_USER` | Пользователь SSH |
| `SSH_PASSWORD` | Пароль SSH |
| `REMOTE_DB_HOST` | Адрес MySQL на удалённой стороне туннеля |
| `REMOTE_DB_PORT` | Порт MySQL на удалённой стороне (по умолчанию `3306`) |
